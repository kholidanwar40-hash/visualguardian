Proses Algoritma

    Algorithm Breakdown

    Bagaimana proses algoritmanya berjalan?

    01

    Caesar Cipher

    Setiap huruf digeser sejumlah nilai tertentu. Huruf Z akan kembali ke A memakai operasi modulo 26.

    encrypted = (index + shift) % 26
    02

    XOR Stream

    Karakter data dan karakter kunci diubah ke kode angka, lalu diproses dengan operasi XOR.

    cipher = charCode ^ keyCode
    03

    Data Masking

    Bagian tengah data diganti simbol sehingga data masih dapat dikenali tanpa membuka seluruh isi.

    an***@example.com
    04

    SHA-256 Hashing

    Data diubah menjadi sidik jari digital sepanjang 64 karakter hex dan tidak bisa dikembalikan ke teks asli.

    hash = SHA256(data)
    05

    Base64 Encoding

    Data dikodekan menjadi format teks yang aman dikirim atau disimpan, lalu bisa didekode kembali.

    encoded = btoa(data)

    Materi Paparan

    Poin yang bisa dijelaskan saat presentasi

    Fungsi Sistem

    Sistem membantu mengamankan data dengan mengubah bentuk asli menjadi bentuk terenkripsi atau tersamarkan.

    Alur Kerja

    Input dipilih, parameter algoritma disiapkan, data diproses, lalu output dan langkah proses ditampilkan.

    Keamanan

    Caesar cocok untuk edukasi, XOR menunjukkan konsep kunci, masking cocok untuk tampilan publik, SHA-256 untuk sidik jari data, dan Base64 untuk encoding teks.

    Keterbatasan

    Project ini untuk demonstrasi algoritma. Pada sistem nyata diperlukan algoritma standar seperti AES dan manajemen kunci aman.
